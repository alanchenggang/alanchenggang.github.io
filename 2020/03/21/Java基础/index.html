使用变量的时候的注意事项：

  1. 如果创建多个变量，那么变量的名称不可重复；
  2. 对于float 和 long 类型来说，字母后缀F和L不可丢掉(注意全部用大写)；

  5. 变量的使用不能超过作用域的范围 ；
    作用域：从定义变量的一行开始，一直到直接所属的大括号结束为止。（先创建后使用）
  6. 可通过一个语句创建多个变量，但一般情况下不推荐这么写；

数据类型转换

当数据类型不一致时，将会发生数据类型转换；
==字节数与数据范围不相关==

自动类型转换：（隐式转换）

  1. 特点：代码不需要进行特殊处理，自动完成；
  2. 规则：范围从小到大；

强制类型转换：（显式转换）

  1. 代码需进行特殊的格式处理，不可自动完成
  2. 格式： 范围小的类型 范围小的变量名 = (范围小的类型) 原味范围大的数据

优先级问题

小括号当中的表达式总是优先计算 以多用小括号来代替运算符优先级的定义；

自增自减运算符:

  1. 在单独使用的时候，前++和后++并无区别
  2. 在混合使用的时候，有==重大区别==
    A.如果是==前++==，那么变量==立刻马上+1==，然后拿着结果进行使用；==先加后用==
    B.如果是==后++==，那么优先使用变量本来的值，==然后再让变量+1==；==先用后加==

逻辑运算符

  1. 与(并且)：全都为true才为true 否则为false
  2. 或(或者)：至少有一个true才为true 全都是false才为false
  3. 非(取反)：本来是true变为false 本来是false 变为 true

与（&&） 或（||）具有短路效应：
如果根据左边的条件即可得知最终结果，则后面的语句不再执行，从而节省一定的性能。

注意事项

  1. 逻辑运算符只适用于boolean 类型的变量
  2. 与、或需要左右各有一个boolean值，但取非仅要一个boolean值即可
  3. 与、或两种运算，如果有多个条件可以连续写

方法总结

  1. 方法应该定义在类当中，但是不能在方法当中再定义方法，不能嵌套；
  2. 方法定义的前后顺序无所谓；
  3. 方法定义之后不会执行，如果希望执行，一定要调用：单独调用、打印调用、赋值调用；
  4. ==如果方法有返回值，那么必须写上“return 返回值；”，不能没有==；
  5. return 后面的返回值数据，必须和方法的返回值类型对应起来；
  6. 对于一个void没有返回值的方法，不能写return后面的返回值，只有return自己；
  7. 对于方法当中最后一行的return可省略不写；
  8. 一个方法当中可以有多个return语句，但是必须保证同时只有一个能被执行到，两个return不能连写；

方法重载(OverLoad)

对于功能类似的方法来说，因为参数列表不一样，却需要记住太多不同的方法名称，为编程带来麻烦，故引入方法重载。

方法的重载：==名称相同，参数列表不同==。

好处：只需要记住一个方法名称，就可以实现类似的多个功能

方法重载与下面因素相关

  1. 参数的个数不同
  2. 参数的类型不同
  3. 参数的多类型顺序不同

方法重载与下面因素无关

  1. 与参数名称无关
  2. 与方法的返回值类型无关//如果参数个数相同，则区分不出来到底调用哪种方法。

  2. 堆内存heap：==凡是new出来的东西，都在堆当中==
    堆内存里面的东西都有一个地址值；16进制
    堆内存里面的数据都有默认值 
    如果是整数类型，那么默认为0；
如果是浮点类型，那么默认为0.0;
如果是字符类型，那么默认为'\u0000';

如果是引用类型，那么默认为null；
数组存放在堆内存当中
  3. 方法区Method Area：存储。class相关信息，包含方法的信息

类和对象

  • 类：是一组相关属性和行为的集合。可以看成是一类事物的模板，使用事物的属性特征和行为特征来描述该类事物。
  • 对象：是一类事物的具体体现。对象是类的一个实例（对象并不是找个女朋友），必然具备该类事物的属性和行为。
  • 属性：是什么
  • 行为：能做什么

类是对一类事物的描述，是抽象的。

对象是一类事物的实例，是具体的。

类是对象的模板，对象是类的实体。

成员变量和局部变量的区别

  1. 定义的位置不一样
    • 局部变量：在方法内部；
    • 成员变量：在方法外部，直接写在类当中
  2. 作用范围不一样
    • 局部变量：只有在方法内才能使用，出了方法就不能使用；
    • 成员变量：整个类全都可以使用
  3. 默认值不一样
    • 局部变量：没有默认值，如果要使用，必须手动赋值
    • 成员变量：如果没有赋值，会有默认值，规则和数组一样
