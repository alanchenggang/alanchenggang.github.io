方法的完整定义格式

方法其实就是若干语句的功能集合。

参数：进入方法的数据，相当于工厂中的原料；

返回值：从方法中得出来的数据，相当于工厂中的成品。

定义方法的完整格式：

修饰符：现阶段的固定写法 public static

返回值类型： 方法产生的返回值的数据类型

方法名称: 方法的名称，规则和变量一样，小驼峰式

参数类型：进入方法的数据是什么类型

参数名称：进入方法的数据对应的变量名称

PS：参数如果有多个使用逗号进行分隔

方法体：方法需要实现的功能，若干行代码

return：两个作用：1.停止当前方法 2.后面的结果数据返回值还给调用处

返回值：方法执行后最终产生的数据结果

注意：return后面的”返回值”必须和方法名称前面的”返回值类型”保持对应

方法的三种调用格式

  1. 单独调用：方法名称（参数）；
  2. 打印调用：System.out.println(方法名称（参数）);
  3. 赋值调用：数据类型 变量名称 = 方法名称（参数）；
注意事项：

此前学习的方法，返回值类型固定写为void，这种方法只能够单独调用，不能进行打印调用或者赋值调用。

方法总结

  1. 方法应该定义在类当中，但是不能在方法当中再定义方法，不能嵌套；
  2. 方法定义的前后顺序无所谓；
  3. 方法定义之后不会执行，如果希望执行，一定要调用：单独调用、打印调用、赋值调用；
  4. ==如果方法有返回值，那么必须写上“return 返回值；”，不能没有==；
  5. return 后面的返回值数据，必须和方法的返回值类型对应起来；
  6. 对于一个void没有返回值的方法，不能写return后面的返回值，只有return自己；
  7. 对于方法当中最后一行的return可省略不写；
  8. 一个方法当中可以有多个return语句，但是必须保证同时只有一个能被执行到，两个return不能连写；

方法重载(OverLoad)

对于功能类似的方法来说，因为参数列表不一样，却需要记住太多不同的方法名称，为编程带来麻烦，故引入方法重载。

方法的重载：==名称相同，参数列表不同==。

好处：只需要记住一个方法名称，就可以实现类似的多个功能

方法重载与下面因素相关

  1. 参数的个数不同
  2. 参数的类型不同
  3. 参数的多类型顺序不同

方法重载与下面因素无关

  1. 与参数名称无关
  2. 与方法的返回值类型无关//如果参数个数相同，则区分不出来到底调用哪种方法。
